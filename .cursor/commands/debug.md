# Debug de Problemas

Investigar e resolver bugs, erros e comportamentos inesperados.

## Processo de Debug
1. Coletar informações:
   - Descrição do problema
   - Logs de erro (se houver)
   - Contexto (quando acontece, quem afeta)

2. Investigar:
   - Reproduzir o problema
   - Analisar código relacionado
   - Verificar dependências
   - Checar dados no banco

3. Diagnosticar:
   - Identificar causa raiz
   - Avaliar impacto
   - Propor soluções

4. Resolver:
   - Implementar correção
   - Testar solução
   - Documentar fix

## Ferramentas
- Logs do Odoo
- PostgreSQL queries
- Python debugger (pdb)
- Browser DevTools (para frontend)
